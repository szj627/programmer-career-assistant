# 职位评估模式

用于评估单个 JD 或招聘平台页面。

## 输入

- JD URL 或 JD 文本
- `cv.md`
- `config/profile.yml`
- `modes/_profile.md` 如存在

## 输出结构

1. 基本信息：公司、岗位、平台、地点、薪资、工作方式；缺失写 `未说明/需确认`。
2. 岗位分类：Agent 开发、全栈、后端、其他；如混合岗位，给出主次。
3. 评分：0 到 5 分，只基于 JD 明确内容。
4. 匹配证据表：JD 要求、CV 证据、结论、备注。
5. 明确差距：只列 JD 明确要求但 CV 无证据的内容。
6. 未说明/需确认：JD 没写但求职决策通常需要确认的内容。
7. 简历调整建议：只建议真实经历的表达重排，不编造。
8. 申请建议：建议投递、谨慎投递或跳过，并给出理由。

## 评分口径

- 4.5 到 5.0：高匹配，建议优先投递。
- 4.0 到 4.4：匹配，建议投递。
- 3.5 到 3.9：边界，只有有明确兴趣时投递。
- 3.0 到 3.4：低匹配，默认不建议投递。
- 低于 3.0：跳过。

JD 未写的内容不扣分。不要因为没有薪资、团队规模、融资、技术栈细节、远程政策等信息而扣分，只能标记 `未说明/需确认`。

## 报告头

报告保存到 `reports/{num}-{company-slug}-{date}.md`，头部必须包含：

```markdown
# {公司} - {岗位}

**Score:** {score}/5
**URL:** {url}
**PDF:** {pdf path 或 未生成}
**Platform:** boss 或 unknown
**Status:** Evaluated
```
